如何在p_query中使用复杂的sql语句

Jul 21, 2017 | Hits

p_query 是 Tiptop GP5.0 以上系统提供的简单报表制作工具，只要通过 SQL 就能写出简单的报表内容。

p_query 对稍微复杂点 SQL 的支持度还是不够理想，比如有 MAX，MIN，UNION 等关键字时，就无法正确解析出来。

其实还是有办法的： 将一个复杂的 SQL 在数据库中建成 VIEW ，然后在 p_query 中直接从该 view 中查出数据出来就可以了。

下面讲一个例子：

SELECT DISTINCT ogb31,ogb32,MAX(oga02) oga02
FROM ogb_file,oga_file
WHERE ogb01=oga01 AND oga08<>1 AND oga09 in (‘2’,’3’,’4’,’6’)
GROUP BY ogb31,ogb32
ORDER BY ogb31,ogb32

上面这段 SQL 是要找出订单的最晚出货时间。
如果直接在 p_query 里面写这样的 SQL，max(oga02)那个会有解析的问题，而且 p_query 画面上也无法下查询条件。
如果把上面这段 SQL 写成 view 就很好的解决这个问题了：

CREATE OR REPLACE VIEW delivery_date AS
SELECT DISTINCT ogb31,ogb32,MAX(oga02) oga02
FROM ogb_file,oga_file
WHERE ogb01=oga01 AND oga08<>1 AND oga09 in (‘2’,’3’,’4’,’6’)
GROUP BY ogb31,ogb32
ORDER BY ogb31,ogb32;

然后直接在 p_query 里面写 ：

SELECT * FROM delivery_date

可以起到一样的查询效果，而不会有任何问题。

【注意】 VIEW 需要在每个营运中心对应的数据库中都建一下，否则会导致没有建该 VIEW的营运中心执行此报表失败。
